### 💡 代码原理解析总结 (帮你更好理解代码逻辑)：
1. **这是典型的 `Tool Use` (函数调用) Agent**：代码的核心在于 `agent_loop` 函数。模型不知道你的本地环境是什么样，但你给了它一个 `bash` 工具。
2. **多轮思考闭环**：当你说 "帮我看看当前目录下有哪些 Python 文件" 时：
   - 模型思考 -> 决定调用 `bash` 工具，参数为 `ls *.py`。
   - 代码捕获到意图 -> `run_bash` 本地执行 `ls *.py` -> 拿到结果。
   - 代码把结果塞回 `messages` 再次发给模型。
   - 模型看到结果 -> 总结回答 -> `stop_reason` 不再是 `tool_use` -> 循环结束，输出最终结果给用户。
3. **安全提示**：由于 `subprocess.run` 使用了 `shell=True`，这个脚本具有极高的系统权限（可以删除文件、安装软件等），虽然代码里加了一个简单的 `dangerous` 列表拦截，但**千万不要在生产服务器或含有机密数据的系统上以 root 权限随意运行**，最好在虚拟机或 Docker 容器内测试。
