# AI Coding Agent — 项目经历（简历素材）

> 以下两个版本描述同一项目，按简历空间选择使用。

## 版本一：详细版（9 条 | 适合项目经历栏空间充裕时使用）

> **AI Coding Agent — 从零构建类 Claude Code 的生产级智能编程助手**
> *Python / Anthropic SDK / Multi-Agent / MCP 协议 ｜ 独立开发，20 个渐进模块，最终集成 2283 行代码*

- 设计 **7 步流水线 Agent 核心循环**作为统一执行框架，将任务注入、上下文管理、工具组装、权限校验等环节拆分为独立阶段；采用 dispatch-map 分发架构与 **4 类事件钩子**（输入提交/工具执行前/工具执行后/会话终止）将横切关注点完全解耦，最终集成 **27 个内置工具**，新增工具或安全规则无需修改核心循环代码

- 针对单 Agent 长时间工作易偏离目标、多 Agent 协作缺乏统一视图的问题，设计 **两层计划系统各司其职**：会话级计划（内存存储，超 3 轮无更新自动提醒聚焦）解决注意力漂移，项目级任务图（文件持久化 + 依赖关系 DAG + 多 Agent 认领机制）支撑团队协作，两层并行运行互不干扰，兼顾"当前做什么"和"项目整体进度"

- 为实现复杂任务的高效分解，从实际需求出发抽象出 **两种委托模式**：同步子智能体（独立上下文隔离 + 安全轮次上限 + 仅返回文本摘要）适用于计算密集型子任务，异步协作智能体（后台线程常驻 + 文件邮箱多轮通信 + 空闲自动扫描并认领新任务 + 计划审批流程门禁）适用于长期并行协作，覆盖从单次委托到持续协作的完整需求频谱

- 为避免 Agent 自主执行代码造成不可逆破坏，构建 **三层递进式安全防线**（指令黑名单硬拦截 → 正则规则匹配 → 用户交互确认），实现路径遍历防护防止越权读写文件，危险命令列表同步覆盖 **Linux/Windows 双平台共 12 类高危操作**（如 rm/del、mkfs/format、sudo/runas），安全规则以事件钩子形式外挂，新增防护策略不影响已有工具逻辑

- 面对 LLM 上下文窗口的硬限制与 API 按 Token 计费的成本压力，建立 **4 层渐进式上下文压缩管道**（大结果自动落盘 → 超量消息截断保留头尾 → 旧结果占位符替换 → LLM 摘要压缩），前 3 层为纯字符串操作零额外成本，仅第 4 层触发 API 调用；配合 **长期记忆系统**（对话结束自动提取关键信息并持久化，≥10 条触发自动去重合并）与 System Prompt 确定性缓存（上下文不变时复用缓存键保持 API 缓存命中）、技能目录轻量注入（约 **100 Tokens 替代 2000 Tokens，节省 95%**），将超长会话的 Token 消耗降低约 **60%**

- 针对慢命令阻塞交互、周期任务依赖人工触发两个痛点，实现 **两种互补的异步机制**：后台任务模块（守护线程执行 + 自动识别构建/测试/安装等慢命令 + 完成主动通知主循环）使耗时操作不再阻塞用户交互；定时调度器（支持标准 5 字段 Cron 表达式 + 文件持久化防重启丢失 + 到点自动触发执行）支撑"每日定时巡检"等周期场景，通过互斥锁确保人工交互与自动执行互不冲突

- 为解决 Agent 工具集固定不可扩展的局限，搭建 **MCP 风格插件架构**，支持运行时动态连接外部工具服务器、自动发现其工具列表并合并到统一工具池（命名空间隔离防止名称冲突），实现第三方工具服务器的热插拔集成，接入新工具服务器无需重启 Agent

- 面对 LLM API 的三类常见故障（输出截断/上下文超限/服务限流），建立 **三条独立恢复路径**：输出截断时自动扩容 Token 上限并注入续写指令（最多 3 次续写），上下文超限时触发紧急压缩降级重试，遇服务限流或过载时执行指数退避重试（**500ms 起始 × 2 倍递增 + 25% 随机抖动，最多 3 次**）+ 连续失败自动切换备用模型，保障 Agent 在弱网和高负载场景下的可用性

- 面对多线程并发执行的竞态风险，设计 **5 种并发角色的线程模型**（主交互线程 / 定时调度守护 / 自动执行线程 / 后台任务线程 / 协作智能体线程），按职责划分临界区并施加分层锁保护，实现线程安全的终端输出确保用户输入行不被后台日志打断，全系统 **4 个守护线程**并发运行稳定无死锁
